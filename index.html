API endpoints del Worker

Método Ruta Trigger Función
POST /api/capture-lead Formulario Guarda lead, crea cola de 7 emails, envía email 1, Meta CAPI
POST /api/send-emails Cron (cada hora) Revisa email_queue, envía pendientes via Resend, actualiza status
GET /api/unsubscribe Link en email Marca lead como unsubscribed, cancela emails pendientes
POST /api/email-webhooks Resend Procesa opens, clicks, bounces; cancela emails en hard bounce
GET /api/dashboard Dashboard Retorna overview, leads por procedimiento, por dominio, recientes

Esquema de base de datos (D1 / SQLite)

TablaCampos clavePropósito
doctors id, slug, name, phone, instagram, city Tenants — multi-doctor ready. Dr. Marcos es el primero.
leads doctor_id, name, email, phone, procedure, source_domain, language, utm_*, status Cada persona que llena un formulario. Status: new → nurturing → contacted → booked → converted
email_queue lead_id, procedure, sequence_number (1-7), scheduled_at, status, resend_id Cola de emails programados. 7 por lead, días 0/3/5/7/10/14/21
events lead_id, event_type, metadata (JSON) Log de actividad: form_submit, email_sent, opened, clicked, bounced, unsubscribed
conversations lead_id, channel, role, message Historial del chatbot (WhatsApp, Instagram, Facebook)
05

Componentes del sistema

Landing pages (8 procedimientos)

  • /liposuccion/ — Liposucción
  • /bbl/ — Brazilian Butt Lift
  • /aumento-busto/ — Aumento de Busto
  • /abdominoplastia/ — Abdominoplastia
  • /rinoplastia/ — Rinoplastia
  • /mommy-makeover/ — Mommy Makeover
  • /rejuvenecimiento/ — Rejuvenecimiento Facial
  • /verifica-cirujano/ — Verifica tu Cirujano

Contenido por procedimiento

  • Landing page con hero, formulario, credenciales, FAQ
  • Guía HTML (~10 páginas educativas)
  • PDF descargable (mismo contenido)
  • Página de gracias con CTA WhatsApp
  • 7 emails de nurturing (21 días)

Email sequences via Resend

  • Dominio verificado: cirugiaplasticacertificada.com
  • SPF + DKIM configurados en Cloudflare
  • 56 emails (8 procedimientos × 7)
  • Webhooks para opens, clicks, bounces
  • Hard bounce cancela emails restantes
  • Unsubscribe en cada email

Meta tracking (dual)

  • Browser: Pixel JS en cada landing page
  • Server: CAPI desde Worker en cada lead
  • Deduplicación via event_id compartido
  • Datos hasheados SHA-256: email, tel, nombre
  • Custom data: procedimiento, dominio, idioma

Chatbot IA (ManyChat + webhook)

  • WhatsApp Business + Instagram + Facebook
  • ManyChat recibe → webhook a Worker
  • Worker envía a Claude/GPT con system prompt
  • Personalidad profesional, reglas inquebrantables
  • Captura gradual de datos del paciente
  • Protocolo de emergencias médicas

Seguridad y compliance

  • SSL/TLS automático via Cloudflare
  • CDN global, DDoS protection
  • Aviso de Privacidad LFPDPPP
  • Derechos ARCO documentados
  • CAN-SPAM: unsubscribe en cada email
  • Datos de Meta hasheados (SHA-256)
